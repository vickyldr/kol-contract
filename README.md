# KOL 合同助手

一个纯前端网页工具，帮 KOL 商务同学：

1. **合同填充** —— 选产品、选模板（或上传模板）、填红人合同信息，一键生成合同。**模板无需改动**：系统按 `标签:` 自动定位空位填入，并填好单价 / 视频数量、把正文里的 `【Rythmix】` 换成所选产品名。付款字段随模板自动识别（IBAN / SWIFT / PayPal / Payoneer）。
2. **合同修改** —— 按内部 SOP 辅助改约：红人原文 → AI 整理给 TL → TL 回复 → AI 翻译给红人 → 一键生成**修改处黄色高亮**的合同，下载后截图给红人。
3. **按红人 + 产品归档** —— 同一红人同一产品的合同（含填充信息与全部修改轮次）存为**一条记录**，支持搜索。

内置了你的 8 套模板（英文 bank-IBAN / bank-JP / bank-SWIFT / paypal / paypal-prepay / payoneer、日文框架、韩文框架），开箱即用。所有数据只存在你自己的浏览器（`localStorage`），不上传任何服务器；AI 调用直连 Anthropic（需自备 API Key）。

## 运行

```bash
npm install
npm run dev      # 本地开发，浏览器打开终端里给出的地址
npm run build    # 产出静态文件到 dist/，可部署到任意静态托管
npm run preview  # 预览构建结果
```

## 使用流程

1. **产品**：先在「产品」页添加产品（如 Rythmix）。产品名会替换合同正文里的 `【Rythmix】`。
2. **合同记录 → 新建记录**：
   - 选产品、选模板（语言自动识别）。
   - **① 一键智能填充**：把红人给的所有信息（姓名、链接、单价、银行/PayPal 收款信息…）一股脑粘贴进去，AI 自动识别并填到对应栏位；缺失或对不上的会列出来问你，补充后再点一次或手动改即可。
   - 填红人合同信息：法人姓名、社媒账号、KOL 链接、发布平台、单价、合作视频数量；付款字段会**根据所选模板自动出现**（如 bank 模板出现 IBAN / SWIFT / Bank Name，paypal 模板出现 Paypal Name / Email）。
   - 付款信息可选「**本人账户 own account / 第三方账户 third-party**」，自动填入对应的账户栏。
   - 银行模板的注册地址会拆成**街道 / 城市 / 省州**三格填写，自动合并进 Registered Address。
   - 银行账户**币种 Account Type**（如 USD）会写入并覆盖模板里的示例（USD / Turkey Lira 之类）。
   - 「**是否预付**」：默认发布后支付；选预付会把合同里的 Time of Payment 整句替换成预付条款（默认 50/50），可填中文备注让 AI 改写（写清两次支付的时间节点和各付百分之几），也可手动编辑。
   - 填「**红人国家/地区**」用于制裁名单与地区规则检查。
   - 生成前会显示「**检查与提醒**」，按支付 FAQ 自动校验：
     - **OFAC 制裁名单**：国家在名单上（中国大陆/香港除外）→ 红色提示原则上不合作；
     - **PayPal 币种白名单**（仅美/欧/英/港/台/日元，韩币等不支持）；
     - **台湾银行只能收美金**（非 USD 会提醒）；
     - PayPal/Payoneer 账号邮箱格式、Payoneer 不能预付/<$50、银行信息齐全度、SWIFT 格式、户名英文要求、日本支行；
     - 收款人姓名与签约名对不上 → 提醒走第三方账户；
     - **对公账户**：自动识别公司名（Ltd/Inc/有限公司…）或手动勾选，合并成一条提醒（先问求证、建议对私、坚持对公需资质+发票、附我方公司信息与对应发票模板）。
   - 点「一键生成并下载合同」。
3. **合同修改**（红人有修改意见时，在记录里「新建修改轮次」）：
   - 第 1 步：粘贴红人原话 → AI 翻译并按「一点一点」整理成中文，复制给 TL。
   - 第 2 步：粘贴 TL 回复 → AI 生成红人语言的礼貌回复，复制给红人。
   - 第 3 步：AI 按「最小改动原则」给出 `原文 → 修改后` 列表（可手动增删改）→ 一键生成合同，**修改处黄色高亮**（基于本记录当前填好的合同，无需重新上传），下载后发 TL 复核、截图给红人。

记录与修改轮次自动保存；顶部搜索框可按红人姓名 / 产品 / 链接 / 账号 / 模板搜索。

## 模板

- **内置模板**开箱即用，无需在模板里加占位符。
- 也可在「模板」页**上传**自己的 .docx（同样的 `标签: 空值` 表格结构即可自动识别填充）。

填充原理：表格里「标签单元格有文字、相邻单元格为空」的行被识别为可填字段；只填空位、不动甲方（Party A）已有内容；单价 / 视频数量按 `【 】` 锚点填；产品名与支付方式按正文标记替换。

## 产品

内置默认产品：Rythmix、VivaVideo、AICatch、VivaCut、WiseMeal、Recco、Inspo、AIFlow、Rymo，可在「产品」页自行增删改。产品名会替换合同正文里的 `【Rythmix】`。

## 部署（Vercel）与内置 Key

1. Vercel → New Project → Import 本仓库 → Framework 选 **Vite** → Deploy。
2. 想让同学**免填 Key**：在 Vercel 项目的 **Settings → Environment Variables** 添加
   `VITE_QWEN_KEY = <你的千问 Key>`（用 Claude 则 `VITE_ANTHROPIC_KEY`），保存后 **Redeploy**。
   之后 App 会自动用这个 Key，**同学打开即用、无需任何配置**（网页里已没有填 Key 的入口）。
   只设 `VITE_ANTHROPIC_KEY`（不设 `VITE_QWEN_KEY`）则自动切到 Claude。

   > **千问 Key 的国内/国际之分**：阿里云"国内百炼"(aliyun.com) 和"国际站 Model Studio"(alibabacloud.com)
   > 是两套独立账号，Key **不通用**，用错会报 `401 invalid_api_key`。本工具**默认走国内站**
   > (`dashscope.aliyuncs.com`)。若你用的是**国际站** Key，加一个环境变量 `VITE_QWEN_REGION = intl`
   > 即可切到国际站；也可用 `VITE_QWEN_BASE_URL` 完全自定义接口地址。

> ⚠️ 这是纯前端应用，内置 Key 会随网页打包、能在浏览器里被看到——仅适合**内部使用**。请给该 Key
> 设置**消费上限**，泄露时在阿里云百炼控制台**轮换**即可。Key 只存在 Vercel 环境变量里，不进 Git 仓库。
> 本地开发可在项目根目录建 `.env.local` 写 `VITE_QWEN_KEY=...`。

## AI 服务商

「合同修改」的翻译 / 总结 / 改约：

- **通义千问 Qwen（默认）**：`qwen-flash`，约 $0.05 / 百万输入 token、$0.40 / 百万输出，一次改约（3 次调用）通常不到 1 分钱。用国际站 `dashscope-intl` 直连。
- **Claude（Anthropic）**：`claude-opus-4-8`，质量更高、更贵。

Key 仅存在本浏览器，请求直连服务商，不经过任何中间服务器。

## 技术栈

Vite + React + TypeScript；`pizzip` 解析 .docx，直接改写 `word/document.xml`：标签填充、`【 】` 锚点填充、正文标记替换，以及给修改文字加 `<w:highlight w:val="yellow"/>` 高亮，均能正确处理被拆成多个 run 的文字片段。
